Pseudoclassical Inheritance

  • Function objects inherit from the Function.prototype property.
  • Objects can inherit properties and methods from other objects using the prototype property.
  • For a complete implementation, see John Resig's Simple JavaScript Inheritence.
function Animal(name) {
    this.type = "animal";
    this.getName = function() { return this.name };
}
function Cat(name) {
    this.name = name;
}
Cat.prototype = new Animal();

var zoe = new Cat("Zoe");
console.log(zoe.getName()); // logs "Zoe"

Doctypes

  • A Tag that tells the browser what HTML or XHTML specification the Web page uses.
  • Helpful for making sure browsers render your Web page in Standards mode (instead of Quirks mode)
  • It's always put at the very top of an HTML document.
  • It's basically safe to make full use of the HTML5 doctype now: 
    <!DOCTYPE html>

Primitives vs. Constructors

  • Primitive types, in JavaScript, are built-in, basic types, which have special chracter sequences to define them.
    • e.g., [], {}, "", true, and false are all primitives.
  • Most primitives also have Constructors, which are a formal way of creating objects. Concstructors make use of the new operator we looked at with Functions.
    • e.g., new Array(), new Object(), new String(), new Boolean(true), and new Boolean(false), are all constructors for basic JavaScript types.
  • Typically, there is little reason to use the constructor method, as primitives are built-in and shorter.

Gotchas

  • If you do feel inclined to use the constructors, pay attention to this Gotcha: 
    var primative_array = [7]; // This represents an array with one element, containing the number 7.
var constructor_array = new Array(7); // This represents an array with seven elements, all set to undefined.
var constructor_array_several = new Array(7,8,9); // This represents an array with three elements, set to 7,8 & 9 respectively.
  • Also be aware that, because everything is an Object in JavaScript, primatives have properties and methods as well: 
    "Primatives are objects, too!".length; // Returns 28
"Mozilla Firefox".toLowerCase(); // Returns "mozilla firefox"

Document Object Model

  • A "document" refers to any XML or HTML.
  • For our Web purposes, a "document" is a Web page, including:
    • History (Back/Forward)
    • All HTML Elements
    • Styles
    • Web Browser Window!

What's the object model?

  • The Document Object Model (or DOM) is an API (Application Programming Interface).
  • An API is way for two different technologies (here: HTML and JavaScript) to interact.
  • In JavaScript, the document is implmented using JavaScript Objects: 
    window.history.back();
document.getElementById('picture');
document.getElementById('picture').style.border = "1px solid red";

Web Browser Gotcha of the DOM: window

  • The window and document objects appear to be wholly separated, but this is not true.
  • By calling window.document.getElementById('picture');, you'd get the same result as document.getElementById('picture');.
  • For ease of use, modern web browsers assume you'll be using the window object of the DOM.

Events

  • Events are the beating heart of any JavaScript application.Quirksmode.org
  • In order for there to be a two-way interaction between your scripts and your users, you must rely on events.
  • When the user does nearly anything in the browser and when the web page loads or unloads, an event is fired.
  • Back when Netscape first implemented what would become JavaScript, event handlers were attributes you placed on elements in your Markup: 
    <a id="aTag" href="#" onclick="alert('You clicked me!')">Click me!</a>
  • These days, we create event handlers (which are usually functions), in our <script /> tag: 
    var link = document.getElementById('aTag');
link.onclick = function() { alert('You clicked me!'); };
  • Cross-browser compatibility issues really start to show up with event handling, so make sure to test in all your target browsers.

Event Handlers

  • Event handlers are functions that get called when an event takes place (or is "fired").
  • Information about the event is passed to the event handler in different ways between IE & every other browser: 
    link.onclick = doSomething;
				
function doSomething(e) {
	if(!e) var e = window.event; // Detect if the event info was passed through an argument, or needs to be set through window (as is the case with IE)
	alert(e); // alerts information about the event
	return false; // stops the <a> tag from following the href link
}

Using this in Event Handlers

  • this is the special variable that is automatically set, depending on how a function was invoked.
  • In event handlers, this contains a reference to the HTML Element that caused the event handler to fire. 
    link.onclick = doSomething;
				
function doSomething(e) {
	if(!e) var e = window.event;

	this.innerHTML = "I've been affected by an event!";
	return false;
}
