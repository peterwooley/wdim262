Using Git

  • This term, we'll be using the version control software git.
  • Revision control (or version control, source control, source control management, source code management) is the management of changes in files.
  • A Typical revision control system will allow you to compare, restore, and merge various files.
  • The simplest form of revision control is storing backups of your files as you make changes (e.g., index_01.html, index_02.html, index_03.html, etc.)
  • Although the project must be open source to be free, GitHub is a great solution.
  • We'll create an SSH key, sign up for an account, create a repository, then figure out how to actually use it.

Getting Git Up and Running

  • Install git on your platform:
    • Windows: msysgit
    • OS X: Git for OS X
    • Linux: git
  • Open git on your platform:
    • Windows: Open Start » Git » Git Bash
    • OS X: Open Terminal.app
    • Linux: Open a shell
  • Follow along with setting up an SSH key.

Test Git Repository

  • Clone a copy of the repository 
    git clone http://github.com/username/projectName
  • Change directory into the newly-created working copy and add a new directory 
    cd projectName
mkdir trunk
  • Check the status of the working copy 
    git status
  • Add a README file to the folder
  • Add the new file to the working copy 
    git add README
  • Commit the changes to the repository 
    git commit -m "Added READDME."
  • Push the commited changes up to GitHub. 
    git push remote master

Variables in JavaScript

  • You can store different types of information in variables (Numbers, Strings, Objects, etc.)
  • They often start with the var declaration.
  • Then followed by the name of the variable
    • Variable names are case-sensitive
    • Must begin with a letter, $ or _.
  • Then followed by the assignment of the variable's value after an = sign.
  • And finished off with a semi-colon;
var theNumber = 42;
var sentence = "This is a sentence. Note the surrounding quotes.";

What JavaScript Is

  • JavaScript (not Java)
  • Client-side (not Server-side)
  • Interpreted (not Compiled)

Interpreted Languages

What could some of the benefits be of using an Interpreted language?

Interpreted Language Breakdown

  • Pros
    • Multi-platform
    • Indirectly Executed by an Interpreter
    • Smaller file size
    • Always Editable
  • Cons
    • Requires an Interpreter
    • Always Editable

What can it do?

  1. It can count.
  2. It can remember.
  3. It can interpret.
  4. It can run Quake II.
  5. It can run a wicked side-scroller.
  6. It can, but shouldn't, animate a clock that follows your cursor.

What can't it do?

  1. It cannot access files on your computer directly, for safety reasons.
  2. It cannot directly create or modify files on your Web site, as it runs on the client-side.
  3. It cannot be completely trusted to be running, because most every browser lets you turn off JavaScript.
