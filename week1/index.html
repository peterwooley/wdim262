Variables in JavaScript

  • You can store different types of information in variables (Numbers, Strings, Objects, etc.)
  • They often start with the var declaration.
  • Then followed by the name of the variable
    • Variable names are case-sensitive
    • Must begin with a letter, $ or _.
  • Then followed by the assignment of the variable's value after an = sign.
  • And finished off with a semi-colon;
var theNumber = 42;
var sentence = "This is a sentence. Note the surrounding quotes.";

Using Variables

<!DOCTYPE html>
<html>
	<head>
		<title>JavaScript: Hello, World!</title>
		<script>
			var hello = "Hello, World!";
			
			// Create a dialog box containing the value
            // of the hello variable.
			alert(hello);
			
			// Write the value of the hello variable
            // into the document (page).
			document.write(hello);
		</script>
	</head>
	<body>
    	<p>I've written a Hello, World script in JavaScript.</p>
	</body>
</html>

What JavaScript Is

  • JavaScript (not Java)
  • Client-side (not Server-side)
  • Interpreted (not Compiled)

Interpreted Languages

What could some of the benefits be of using an Interpreted language?

Pros

  • Multi-platform
  • Indirectly Executed by an Interpreter
  • Smaller file size
  • Always Editable

Cons

  • Requires an Interpreter
  • Always Editable

What can it do?

  1. It can count.
  2. It can remember.
  3. It can interpret.
  4. It can run Quake II.
  5. It can run a wicked side-scroller.
  6. It can, but shouldn't, animate a clock that follows your cursor.

What can't it do?

  1. It cannot access files on your computer directly, for safety reasons.
  2. It cannot directly modify files on a server, when it runs on the client-side.
  3. It cannot be completely trusted to be running, because most every browser lets you turn off JavaScript.

Debugging JavaScript

  • Add this bit of JavaScript to a page: 
    console.log("I'm sending a message to the Console!");
  • Reload the page.
  • Open The Developer Tools
    • In Firefox, open Firebug (F12, or a little bug in the bottom-right or top-right corner).
    • In Chrome or Safari, open Web Inspector (in Develop Menu).
    • In Internet Explorer, open F12 Tools (with F12).
  • Rejoice!
